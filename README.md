# Ontology Chooser — site folder

A small static website that lets visitors filter 14 categorial schemes (Plato through Glasersfeld) by stance, philosophical commitments, and practical tooling. The whole site is one HTML file plus a PDF — no server, no database, no build pipeline required to run it.

This README explains what's in the folder, how to deploy it, and how to update it.

---

## What's in this folder

| File | What it is | Do you ever edit it? |
|---|---|---|
| `index.html` | The chooser website itself. Self-contained — everything is inside this one file. | **No.** This is generated by `build.py`. |
| `ontology_descriptors.json` | The data: descriptors for all 14 ontologies (stance, badges, commitments, tooling). | **Yes.** This is the file you edit when you want to change the data. |
| `ontological-categories-v9.pdf` | The full diagrammatic deck. The "view the full deck" link in the chooser opens it. | Replace it if you regenerate the PDF. |
| `diagrams/diagram-<key>.pdf` | One single-page PDF per ontology, generated by splitting the full deck. The "view diagram →" links on each result card open the relevant one. | **No.** These are generated by `split_pdf.py`. |
| `build.py` | A short Python script that takes `ontology_descriptors.json`, injects it into `_template.html`, and produces `index.html`. | No. |
| `split_pdf.py` | Splits the full deck into per-ontology PDFs in the `diagrams/` folder. Run after replacing the source PDF. Requires `pypdf`. | No. |
| `_template.html` | The HTML/CSS/JavaScript template (with a placeholder where the JSON gets injected). | Only if you want to change the visual design or chooser logic. |
| `_gen_pdf.py` | The Python script that generates the PDF deck. Reference copy in case you want to extend or modify the deck. | Only if you want to add pages or change the diagrams. |
| `ontology_descriptors.csv` | A flat CSV view of the same data, useful for quick auditing in a spreadsheet. | No (or yes — you can edit and round-trip back to JSON, but that's manual). |

Files starting with `_` are kept for reference and updates but are *not* served to visitors. Only `index.html` and `ontological-categories-v9.pdf` matter at runtime.

## How to deploy the site (5 minutes, no account needed)

The simplest path is **Netlify Drop** — drag a folder onto a webpage and get a live URL in seconds.

1. Make sure this folder contains at least `index.html` and `ontological-categories-v9.pdf`. The `_template.html`, `_gen_pdf.py`, and `build.py` files don't need to be uploaded — but it's harmless if they are, they just sit there unused. (Visitors can't see them unless they guess the filenames.)

2. Open https://app.netlify.com/drop in your browser.

3. Drag this entire folder onto the drop area on that page.

4. Wait about 10 seconds. Netlify gives you a live URL like `https://something-random-12345.netlify.app`.

5. That URL is your live website. Share it.

If you want a memorable URL, click "Claim this site" on the result page — it'll prompt you to create a free account and pick a custom subdomain like `ontology-chooser.netlify.app`. You can also set a custom domain later (e.g. `ontology.your-name.com`) but that's optional and requires buying a domain name (~€12/year).

**Other hosting options that work just as well:** GitHub Pages (requires a GitHub account; push these files to a repo), Cloudflare Pages (similar to Netlify). All free for sites this small.

## How to update the data

You'll occasionally want to:

- correct an uncertain `?` cell after checking the source literature
- add a new descriptor or a new ontology
- fix a typo in an ontology's commitments

The workflow is always the same:

### Step 1 — edit `ontology_descriptors.json`

Open it in any text editor (VS Code, Notepad, TextEdit, anything). The structure is straightforward — each ontology has three sections:

```json
"bfo": {
  "title": "BFO",
  "label": "BFO",
  "page": 9,
  "publication_year": 2002,
  "layer1": { "stance": "Mind-independent", ... },
  "layer2": { "universals": "admits", "tropes": "does-not-admit", ... },
  "layer3": { "encoding_format": ["OWL2", "FOL/CLIF"], ... }
}
```

Three-state Layer-2 values: `"admits"`, `"does-not-admit"`, or `null`. The trailing `?` in some values (e.g. `"admits?"`) marks cells I was uncertain about during the initial fill — when you verify them, drop the `?`.

### Step 2 — run the build script

Open a terminal, navigate to this folder, and run:

```
python3 build.py
```

It reads the JSON, validates it parses correctly, and regenerates `index.html`. You'll see:

```
OK  index.html regenerated — 14 ontologies, 55.1 KB
```

If the JSON is malformed (missing comma, unmatched brace), the script tells you exactly which line and column has the problem and refuses to overwrite `index.html` until you fix it. Your live site never breaks from a typo.

### Step 3 — test locally

Double-click `index.html` to open it in your browser. Confirm the changes look right — filter counts update, badges render, your edits show up where expected.

### Step 4 — re-upload to Netlify

If you used Netlify Drop, drag the folder onto the drop zone again — it overwrites the previous version, same URL, no downtime.

If you used a Netlify account or GitHub Pages, the workflow is slightly different (drag-drop replacement vs. Git push), but the principle is the same: replace the files, the URL stays the same.

## How to update the deck (PDF)

If you want to add a new ontology page or modify the diagrams:

1. Edit `_gen_pdf.py`. The structure is documented inline — each ontology has a `page_<name>` function, and the build sequence at the bottom controls page order.

2. Run `python3 _gen_pdf.py`. It writes a new PDF (the script's `OUT` line tells you where).

3. Copy that PDF back into this folder, replacing `ontological-categories-v9.pdf`. (Or rename and update the link in `_template.html` — but easier to just keep the same filename.)

4. Run `python3 split_pdf.py` to regenerate the per-ontology diagrams in the `diagrams/` folder.

5. If the new ontology should also appear in the chooser, add a corresponding entry to `ontology_descriptors.json` and run `build.py` again.

## Sanity checks built into the chooser

The chooser includes a small validator that runs automatically when the page loads. It surfaces results in a collapsed `<details>` panel near the bottom labeled *"Data validator (consistency checks)"*. Click to expand and you'll see:

- **Warnings** for inconsistencies (e.g. an ontology marked `Endurantist + Perdurantist` but with `process_event ≠ admits`).
- **Informational notes** for ontologies with many uncertain (`?`) cells worth verifying.

If you make a major change to the data, glance at this panel before redeploying.

## Help — something's not working

**The page is blank when I open `index.html` locally.** Most likely: the JSON failed to parse. Open your browser's developer tools (F12 in most browsers, Cmd+Opt+I on Mac), look at the *Console* tab, and you'll see the error. Often it's an unmatched `{` or a missing comma in `ontology_descriptors.json`.

**`python3 build.py` says "command not found".** You don't have Python installed. On Mac/Linux it's almost always there — try `python build.py` instead. On Windows, install Python from python.org (any version 3.6+) and check the "Add to PATH" box during setup.

**The deployed site doesn't show my latest changes.** Browsers cache aggressively. Hard-refresh the live site: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac). If that doesn't help, the upload to Netlify probably didn't go through — check the Netlify dashboard.

**The fonts look wrong.** The chooser uses Google Fonts (EB Garamond, Fraunces, JetBrains Mono). If you have no internet, or the user's browser blocks Google Fonts, it falls back to system serif and monospace fonts — which still look fine, just less distinctive.
